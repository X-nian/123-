## 一、项目一句话说明

这是一个以“未发送的告白”为入口的互动叙事网页。玩家可以观察另一种人生，但作品不把任何选择判定为错误；最终主题是理解代价、接纳已经走过的人生，并继续向前。

## 三、已经确定的创作原则

1. 玩家始终使用第二人称“你”，不设定性别、外貌或固定身份。
2. 关系对象只称“那个人”“对方”“后来遇见的人”，避免固定男女主阻碍代入。
3. 人生没有标准答案；两条时间线都拥有幸福、遗憾和代价。
4. 普通对白稳定呈现，环境化对白只承载关键隐喻。
5. 并列选项默认同级，不预设推荐答案；只有悬停或键盘聚焦时高亮。
6. 页面默认不能后退，但第十次回望提供一次叙事性的例外。
7. 手机端必须在当前屏幕直接看见关键选择，不依赖内部滚动条。

## 四、当前互动流程

1. 人生档案馆提示存在一条长期未关闭的遗憾。
2. 玩家查看七年前的聊天和未发送告白。
3. “没什么。你早点睡。”逐字出现并消散。
4. 消散完成后才出现：“如果你早一些找到它，这一切会不会真的改变？”
5. 玩家选择查看留下来的生活，或让那句话抵达对方。
6. 发送分支使用单一手机屏幕：告白送达、对方正在输入、两条回复依次出现。
7. 另一种人生仍会面对方向和关系之间的代价。
8. 记忆漏洞场景采用对角构图：左上“今天要不要走慢一点？”，右下“你在等人吗？／不知道……”。
9. 最终选择为“留在现在”或“奔赴过去”，两项均带结果说明。
10. 人生档案馆从手机左侧依次发送哲理性留言并收束故事。

## 五、回望机制

- 普通回望：每次获得不同回应，页面不后退。
- 第 3 次：承认玩家可能正处于痛苦谷底，提醒其不要惩罚过去的自己。
- 第 5 次：说明痛苦不等于选择错误，引导玩家重新关注仍会发生的生活。
- 第 10 次：汇总前述含义，并显示“倒回上一界面”。
- 返回依据是 `storyTrail` 记录的真实浏览路径，因此分支不同也能返回实际上一页。

## 六、视觉与响应式状态

- 主色：蓝绿、暖金和少量橙色，避免低对比灰白字。
- 统一字体：`Microsoft YaHei UI`、`Microsoft YaHei`、`PingFang SC` 和系统无衬线回退。
- 关键按钮带内联基础样式，即使发布环境样式加载顺序异常，也不会退回浏览器默认白色按钮。
- 手机端最后选择页已按 390×844 验证：故事区域可用高度与内容高度均为 770px，无内部滚动。
- 第三幕由 JavaScript 明确控制旧句隐藏、新句显示，避免移动浏览器同时显示两段文字。
- 背景人物剪影和摇晃小人已经从页面结构中删除。

## 八、运行与验证方法

直接打开：

```text
web-preview/preview.html
```

推荐使用本地服务：

```powershell
python -m http.server 4173 --bind 127.0.0.1
```

工作目录设置为 `web-preview/`，然后访问：

```text
http://127.0.0.1:4173/preview.html
```

已经执行的关键检查：

- HTML 页面可正常加载。
- 390×844 手机视口最终选择页无滚动。
- 第三幕旧句隐藏后新句才出现。
- 第 3、5、10 次回望提示均能触发。
- 第 10 次回望可以返回真实上一界面。
- `git diff --check` 通过。
