## 一、项目从哪里开始

这段会话最初围绕“当代年轻人精神状态检测”任务书展开。任务要求参考《人生模拟器制作_附件版》的结构和文风，制作一套可直接放入 Codex 教程的实战章节。

参考资料：

- [当代年轻人精神状态检测任务页](https://tiaozhuxiansheng.com/tasks/mental-state-check/)
- [《人生模拟器制作_附件版》](https://github.com/Mr-Salticidae/codex-tutorial/blob/main/docs/%E4%BA%BA%E7%94%9F%E6%A8%A1%E6%8B%9F%E5%99%A8%E5%88%B6%E4%BD%9C_%E9%99%84%E4%BB%B6%E7%89%88.md)

对任务的核心判断是：它并不只是“补一篇文章”，而可能包含产品制作、过程复现、截图、配图和教程写作。开工前必须确认是否已有项目源码、试玩地址、制作过程与原始素材，否则工作量会被严重低估。

## 二、任务拆解方法

对标附件所隐含的教程结构如下：

1. 用问题或情境快速建立体验动机。
2. 展示试玩视频或成品预览。
3. 用一句话讲清互动流程。
4. 给出第一轮开工提示词。
5. 先审阅产品蓝图，再进入制作。
6. 展示使用占位素材的可运行版本。
7. 说明试玩、截图和反馈方法。
8. 展示一至两个真实迭代案例。
9. 演示如何用自然语言扩充功能。
10. 替换正式美术素材。
11. 选择音效或背景音乐。
12. 打包成可分享版本。
13. 给出进阶功能方向。
14. 用开工准备清单收尾。

因此，收到类似任务时，应先把输入分为三类：

- 已有：源码、HTML/CSS/JavaScript、图片、视频、可运行页面。
- 只需整理：已有功能的截图、教程化表达、素材说明。
- 仍需制作：缺失页面、逻辑、视觉素材、演示视频或部署步骤。

## 三、创作方向的转变

项目随后从“精神状态检测”转向一个更适合小体量制作的互动叙事概念：人生档案馆允许人撤回一次过去的选择。

会话比较了两个方向：

### 方向 A：错过一个人

- 核心遗憾：一条没有发出的消息。
- 主要场景：聊天记录、旧照片、车站和少量回忆。
- 角色较少，分支较短。
- 适合制作成 8～12 分钟的网页互动故事。

### 方向 B：错过一种人生

- 核心遗憾：没有选择另一座城市、专业、职业或生活方式。
- 需要同时建立两套人生状态。
- 文本、场景、素材和分支成本更高。
- 更适合后续扩展为完整互动产品。

最终选择“错过一个人”，因为它只需一条主线、少量角色和集中场景，更容易先完成可玩的最小版本。

## 四、故事核心

暂定名：**《那条没有发出的消息》**

玩家在人生档案馆中找到七年前未发送的一句告白：

> 其实，我也喜欢你。

玩家获得一次撤回“删除消息”的机会，并进入消息成功发送后的时间线。新时间线里，两个人确实拥有过幸福，但这段关系并不是没有代价的完美答案。玩家最终需要决定，是覆盖原人生，还是接受已经发生的人生。

故事的原则不是证明“当初没在一起才是正确答案”，而是让两条人生都真实成立：

- 没有发送消息的人生里，玩家失去了一段可能发生的关系，也获得了只有这条路上才会遇见的人和成长。
- 发送消息的人生里，两个人确实靠近过，但仍会面对城市、职业、关系节奏和个人成长的代价。

最终主题：**选择从来不是标准答案；人真正能做的，是理解代价，接受已经走过的路，并继续向前。**

## 五、最小体验结构

第一版控制为 8～12 分钟，避免发展成大型分支小说：

1. 收到“人生撤回权限”。
2. 打开七年前未发送的旧消息。
3. 选择是否撤回“删除消息”。
4. 查看两个人在一起后的若干片段。
5. 逐渐发现另一条时间线的代价。
6. 再次确认是否覆盖原人生。
7. 接受选择带来的结果。
8. 用哲理性对白收束，并鼓励玩家继续向前。

## 六、不可后退的交互规则

人生不提供真正的后退选项。

- 选择页面可以保留“后退”按钮，以回应用户习惯。
- 点击后不发生页面回退，只弹出提示：**人生没有后退选项。**
- 用户可以关闭提示并继续当前选择。
- 已确认的选择不得通过浏览器返回、键盘快捷键或历史记录被撤销。
- 不提供“提前退出替代人生”的捷径。

这一规则不是单纯限制操作，而是作品主题的一部分：玩家必须带着已经发生的选择继续前进。

## 七、角色前进与页面转场

主角每次选择后都要表现为从左向右前进，但角色本体不改变页面中的固定位置：

- 主角播放原地步行动画。
- 背景、地面纹理、光影和前景物体向左移动。
- 由环境位移制造“角色向前走”的感觉。
- 每段对话结束后，当前页面逐渐淡化、失焦，再淡入下一场景。

整体情绪从温暖、期待，逐步转为灰蓝、沮丧或悲伤，最后进入克制的释然。颜色、声音、动画速度和留白应共同完成情绪变化，避免只依靠对白煽情。

## 八、选择后果动画系统

选择之后不应立即跳页，而要加入一段“动作—错位—等待—落空”的后果动画，让玩家看见选择如何改变两个人的距离。

核心视觉母题在 v0.5 中得到修正：原时间线里，告白从未发送。主角发送的只有“没什么。你早点睡。”

1. 主角在输入框中写下告白，又逐字删除。
2. 周遥看到“没什么”，把它理解为主角最终的回答。
3. 周遥打开键盘又收起，最终关闭聊天框。
4. 主角反复下拉或刷新屏幕，等待的不是告白回复，而是希望周遥再问一次。
5. 两个人都没有拒绝对方，只是分别把一句没有说完的话，当成了对方最终的答案。

这个母题可以在后续场景中不断变形：

- 删除消息时，两个人在现实中错过一次对视。
- 恢复消息后，对方终于打开键盘并开始输入。
- 打开聊天和放下手机使用不同动画反馈。
- “留下”与“前往海城”分别表现距离、沉默和生活节奏的变化。
- 最终选择时，两条时间线从主角身后缓慢退场，而不是被粗暴删除。

手机动作贯穿全片：发送、关闭、刷新、扣下、重新拿起。动作本身逐渐改变含义，让观众意识到：真正令人难受的并非单纯没有收到回复，而是两个人曾经在不同时间靠近。

## 九、对话框的场景化视觉系统

对白不固定在屏幕中央，也不统一使用普通气泡，而是附着在故事世界里的物体上。位置看似随意，实际必须承担第二层含义。

| 对话载体 | 表层功能 | 隐含意义 |
| --- | --- | --- |
| 手机屏幕 | 展示真实聊天 | 真正说出口的话 |
| 路边落叶 | 承载短句或回忆 | 迟到、错过、无人接住的话 |
| 车票与站牌 | 展示选择提示 | 去留及其背后的代价 |
| 玻璃倒影 | 展示内心对白 | 角色不愿承认的真话 |
| 雨伞与水滴 | 展示脆弱表达 | 被别人接住的情绪与需要 |
| 纸箱标签 | 展示生活片段 | 主角正在建立的新生活 |
| 冰饮杯壁、泡沫、外卖小票 | 展示日常对白 | 平凡但真实的幸福 |
| 地面影子与脚印 | 展示无声选择 | 没有说出口的决定 |
| 晨光与灰尘 | 展示结尾句 | 接受、释然和继续向前 |

### 使用原则

- “随意”只体现在载体丰富、位置自然，不代表真正随机。
- 文字必须保持可读性和足够对比度。
- 关键选择不能被装饰物遮挡。
- 同一载体应保持稳定语义，避免前后矛盾。
- 文字出现与消失方式应贴合物体特征，例如落叶翻转、玻璃雾气消散、水滴滑落。

## 十、结局表达

两个最终选择都不应被设计成“错误答案”。每个结局都要加入一段有哲理、但不说教的对白。

统一精神可以概括为：

> 你的选择从来不会出错。它只是带你看见不同的风景，也让你承担不同的代价。加油，继续向前。

如果玩家保留原人生，可以用更克制的方式收束：

> 你没有失去一段本该发生的人生。你只是走完了一段没有与他同行的人生。

如果玩家覆盖原人生，也不应惩罚玩家：那条时间线可以幸福、遗憾并存，同样是一段真实的人生。

## 十一、素材与技术接口

当前阶段不强制接入图片、配音、音乐或视频生成 API。实现时保留可配置接口：

```text
IMAGE_API=
VOICE_API=
MUSIC_API=
VIDEO_API=
```

未接入 API 时，可采用以下降级方案：

- HTML/CSS 聊天气泡和场景排版。
- CSS 原地步行、淡入淡出和背景位移动画。
- SVG 落叶、车票、水滴、玻璃雾气等轻量视觉元素。
- 剪影、渐变、环境粒子代替完整角色插画。
- 本地音效或静音版本，不阻塞基础体验制作。

## 十三、v0.5 结构优化结论

### 时间线规则

人生覆盖后，执行者保留一份无法向现实证明的“观察记忆”。玩家记得自己看过两条人生，但不能用照片、聊天记录或他人记忆证明被覆盖时间线存在过。

这一规则解决了结局中的记忆悖论：玩家能够感受到阿照消失，却无法在新现实中找到阿照。

### 人物补强

- 周遥习惯在重要问题上先说半句话，再等待对方补完。告白成功不会自动修复这个缺点，它会在三年后的城市选择中重新出现。
- 阿照不再只是“原人生有价值”的证明，而拥有“今天要不要走慢一点”“我在楼下”和门边备用伞等具体生活记忆。
- 告白成功后，主角不会立刻为了爱情永久放弃海城，而是将机会延期一年。三年后的分歧是旧问题重新回来，不是编剧临时制造的冲突。

### 选择记忆

小选择不增加大型分支，但记录玩家面对遗憾时的倾向：回避、直面、留恋或放下。倾向会改变后续旁白、环境文字载体和结局末句。

玩家无法改变已经发生的大事件，但可以决定自己以什么方式理解它。

### 动画分级

- 核心动画：消息发送、暂缓海城、再次选择海城和最终确认，控制在 4～8 秒。
- 情绪动画：阿照出现、关系分歧、记忆漏洞和告别，控制在 2～4 秒。
- 普通过渡：只使用 0.6～1.2 秒淡入淡出。

环境化对白只用于关键句；普通对白使用稳定字幕区，防止玩家持续寻找文字。

- 每次点击“回望”都会得到新的回应，并显示回望次数。
- 第 03 幕文字保持正常方向，不使用镜像字幕。

- 电脑端和手机端均可使用；正式背景图片尚未确定，目前使用偏阳光的渐变占位。

## 十五、下一步建议

1. 根据试玩反馈继续校验文字节奏、按钮尺寸、等待时长和选择压力。
2. 确定男女主角和阿照的正式视觉锚点，再生成或绘制场景图片。
3. 将当前渐变占位背景替换为正式图片，同时保留低带宽和减少动画模式。
4. 增加基础环境音、发送提示音和结尾音乐，但不让声音阻塞体验。
5. 将静态预览整合进正式前端工程，再决定是否公开部署。

## 一句话结论

这不是一部教玩家“选对人生”的作品，而是一部让玩家看见选择代价、接受已经发生的人生，并继续向前的互动叙事。

## 十六、v0.6 自审与去角色化优化

本轮试玩反馈指出两个直接问题：画面文字不够醒目；人物姓名、性别和关系设定过密，观众需要先理解“周遥、阿照和主角是谁”，才能进入自己的情绪。这会把原本具有普遍性的“未发送消息”收窄成观看别人故事。

### 新的叙事原则

- 玩家始终是“你”，不预设性别、外貌和固定身份。
- 旧日关系统一称为“那个人”或“对方”，不给出性别代词。
- 原时间线中后来出现的人统一称为“后来遇见的人”，不再承担工具化配角姓名。
- 城市与职业仅在不可替代时出现；普通人生选择改写为“想去的地方”“自己的方向”。
- 保留关系的具体动作，例如等待、删除、刷新、撑伞和告别；减少人物履历解释。

### 文字与画面层级

- 关键标题使用高亮近白色和更明确的字重，不依赖低对比灰字营造气氛。
- 正文、聊天消息和选项提高字号与行距，手机端不得缩小到需要凑近阅读。
- 普通对白回到稳定区域；落叶、车票、玻璃等环境文字只承载关键隐喻。
- 选项仍不默认高亮，只在鼠标悬停或键盘聚焦时显示强调。
- 背景继续保持明亮暖色，但通过文字阴影、半透明深底和边框保证可读性。

### 本轮改写结果

网页预览已移除“周遥”“阿照”等姓名，以及“他/她”式性别指向。两条时间线改为“已经走过的人生”和“那句话发出以后”，结尾选择从技术性的“覆盖/保留”改为更有代入感的“走进没有经历的后来/接住已经走过的人生”。故事仍然保留具体情绪，但把人物轮廓留给玩家自己的记忆来补完。

## 十七、v0.7 可读性与消息动效优化

- 全站统一使用系统无衬线中文字体，重新调整标题、正文、按钮和聊天文字的字号、字重、行高与字间距。
- 背景改为蓝绿与暖金组成的柔和渐变，场景变化保持同一色彩体系，避免文字被杂乱颜色吞没。
- 第三幕改为大尺寸消息方框：“没什么。你早点睡。”逐字出现，短暂停留后模糊消散。
- 发送分支取消左右并列画面，恢复为单一手机屏幕：告白送达后显示“正在输入”，再依次出现两条回复。
- 最终两项选择明确改为“留在现在”和“奔赴过去”，并为每项补充结果说明；默认仍不高亮，只有悬停或聚焦时强调。
- 手机端采用单列滚动布局，缩小装饰物而不缩小核心文字；电脑端保留居中留白与双列信息结构。
- 删除背景中的人物剪影和原地摇晃动画，避免无明确身份的人形元素分散注意力；画面推进只保留背景、光影与转场运动。
